# Notion·to·WordPress 贡献指南

感谢你对 **Notion·to·WordPress** 的兴趣！我们欢迎包括 Bug 报告、功能提议、代码、文档以及翻译在内的所有贡献。

## 如何贡献

### 1. 提交 Bug
- 请使用 [GitHub Issues](https://github.com/Frank-Loong/Notion-to-WordPress/issues) 反馈。
- 提供可复现步骤、期望与实际结果，以及环境信息（WP 版本、PHP 版本等）。

### 2. 功能提议
- 创建带有 `enhancement` 标签的 Issue，详细描述你的想法。

### 3. 提交 Pull Request
- Fork 仓库并创建新分支。
- 遵循 [WordPress 编码规范](https://developer.wordpress.org/coding-standards/wordpress-coding-standards/)。
- 编写清晰的 commit 信息，并在 PR 中说明变更内容与原因。
- 确保代码通过 `php -l`，如可能请添加测试。

### 4. 文档 & 翻译
- 改进 README、Wiki 或新增翻译均可。
- 通过 PR 提交或在 Issue 中提出建议。

### 5. 沟通
- 提问请使用 Issues 或 Discussions。
- 安全问题请私信维护者（见 GitHub profile）。

## 许可
所有贡献将以 GPL v3 或更高版本授权，与本项目保持一致。
